# Congestion Pricing — Visual Story

A spatial + multi-modal investigation of the first months of NYC's January 2025 congestion charge. The site lives in `app/` (Vite + React + scrollama), and is being restructured around a five-act narrative.

## Narrative structure (5 acts)
1. Reframe: total trips up, mode mix tilted toward transit
   ← BEING BUILT (this session)
2. The investigation: rings, why OLS fails, GWR
3. The twist: off-peak stronger than peak
4. Robustness: difference-in-differences + the weekday/weekend null
5. Verdict: the cordon redistributed and tilted, but did not originate

## Design system

- **Background:** paper. **Body text:** ink.
- **Display type (numbers, headers):** Fraunces.
- **Body type:** Inter.
- **Labels and section numbers:** uppercase mono.
- **Color tokens:**
  - `text-decline` — negative numbers, the BT (Bridges & Tunnels) band.
  - `text-rise` — positive numbers, transit bands.
  - `text-ink` — neutral copy.
  - These are vocabulary, not decoration. Don't use raw colors.
- **Italic Fraunces** is reserved for thesis lines under each section number, the bridge sentence at the end of each act, and chart captions. Don't italicize body copy.
- **Charts** inherit the paper background — no white panels.

## Data

- Multi-modal daily trips dataset lives at `public/data/mta_modes.csv`
  with columns: date, mode, count.
- Modes: Subway, Bus, LIRR, MNR, SIR, AAR, BT (Bridges & Tunnels),
  CBD Entries, CRZ Entries (post-2025 only).
- Pre-computed chart data for Act 1 lives at `public/data/act1.json`
  and is generated by `scripts/build-act1-data.ts`. Do not load the
  raw CSV in the browser — always read the pre-computed JSON.
- Treatment date is 2025-01-05.
- Comparison windows are Jan 5 – Apr 30 of each year.

## Things to avoid

- Loading raw CSVs in the browser. Always read the pre-computed JSON written by the build script.
- Raw color classes. Use the semantic tokens above.
- Decorative italics. Italic Fraunces is reserved (see Design system).
- Hardcoded numbers in components. Anything derived from data should come from the build script's JSON output, so the single source of truth is the analysis pipeline, not the JSX.
- Re-smoothing time-series in components. The build script writes the rolled values; components render what they're given.
- Running `npm run build` as a typecheck shortcut. Use `npm run typecheck` (it's faster and doesn't produce artifacts).
